## 이번 진단으로 발견
- ⚠️ **서버 앱 로그가 Cloud Logging에 거의 안 뜸** — uvicorn 접근로그(`INFO: GET ...`)만 보이고, 우리 `logger.info`(genai 생성, `👤 user`/`🦫 beaver` 전사, 경고)는 안 나옴. 통화(calls/10)는 있었는데 전사 로그 0줄. → **로깅 설정 미구성**(app 모듈 로거가 INFO 핸들러 없음). "로그가 안 찍힌다"의 진짜 원인.
- ⚠️ 통화 **재생 속도**: 24k→48k 업샘플 코드는 반영됐으나 사용자가 여전히 "빠르다" → 빌드 반영 여부 + 기기 실제 출력 레이트 확인 필요.
- ❓ "AI가 자꾸 혼자 말함" — 로그가 안 보여 원인 미확정. (에코 피드백 루프 vs 마이크 미입력 의심)

## Part A — 서버 로깅 가시화 (1순위)
1. `main.py` 앱 시작 시 로깅 구성: 루트 로거에 INFO 레벨 + StreamHandler(stdout) 부착(uvicorn 로거와 공존, 포맷 통일). app 모듈 `logger.info`가 Cloud Logging에 나오도록.
   - 과한 라이브러리 로그 억제(httpx/google 등은 WARNING)로 노이즈 최소화.
2. 로컬에서 `logger.info` 출력 확인 → 재배포.
3. 검증: 재배포 후 startup에 `normalcall: Vertex genai 클라이언트 생성...`, 통화 시 `👤 user:` / `🦫 beaver:` 가 실제로 보이는지.

## Part B — AI 대화 속도 확정
1. 앱이 **업샘플 수정 반영된 빌드**인지 확인(핫리스타트/clean 재빌드).
2. 여전히 빠르면: 기기 실제 출력 샘플레이트가 48k가 아닐 수 있음.
   - 임시 진단 로그: 선택된 재생 레이트 + 첫 청크 크기 출력.
   - 근본 수정: 플랫폼 채널로 기기 네이티브 출력레이트 쿼리 → 24k→네이티브로 업샘플(48k 하드코딩 제거).
3. "AI 혼자 말함": Part A로 로그 보이면 `👤 user` 전사 유무로 판정.
   - user 전사 있음 + AI 끊임없음 → 에코/피드백 루프(스피커→마이크) → barge-in/에코 처리 점검.
   - user 전사 없음 → 마이크 입력이 서버에 안 감 → 송신 경로 점검.

## Part C — 대화 기록(call history) 기능
기존 플랜 `docs/20260625_1835_records-bookmarks-plan.md` 기반.
- 백엔드: `GET /calls`(목록)·`GET /calls/{id}`(상세) 준비됨. (필요시 result 문장에 evaluation 노출 추가)
- 프론트: 기록 리스트(캐릭터·날짜·시간·요약·평점) → 상세(분석화면 재사용) → 삭제(스와이프). Riverpod 페이지네이션/새로고침.
