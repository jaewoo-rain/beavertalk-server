2. 도메인 계층 구조 (4-Layer)

각 도메인은 Router → Service → Repository → Model 로 동일하게 나뉜다. (Spring 의 Controller/Service/Repository/Entity)

규율: Service 만 db.commit() 호출(트랜잭션 경계). Repository 는 commit 하지 않음. DTO 는 schemas/ 의 Pydantic 모델.

3. 인증 흐름 (Supabase Auth 위임)

로그인/비번은 Supabase 가 담당. 앱은 access token 을 검증하고 auth UUID 로 member 를 찾거나 자동 생성.

sequenceDiagram
    autonumber
    participant C as 📱 클라이언트
    participant A as FastAPI (deps)
    participant SA as Supabase Auth
    participant MS as MemberService
    participant DB as Postgres

    C->>A: 요청 + Authorization: Bearer <token>
    A->>SA: verify_token(token)
    SA-->>A: AuthUser(uid, email)  또는  None→401
    A->>MS: find_or_create_by_auth(uid, email)
    MS->>DB: SELECT member WHERE auth_user_id = uid
    alt 기존 회원
        DB-->>MS: member
    else 신규
        MS->>DB: INSERT member (기본 캐릭터 지급, onboarding=false)
    end
    MS-->>A: member (CurrentMember 주입)
    A-->>C: 200 (핸들러 실행)

    Note over C,DB: 탈퇴 = Supabase auth 삭제 + member 소프트삭제
(deleted_at 기록, email·auth_user_id NULL → 재가입 허용)

참고: core/supabase_auth.py, core/deps.py, domains/account/service/member_service.py

4. 실시간 통화 → 분석 → 복습 (normalcall)

WebSocket 으로 Gemini Live 와 실시간 통화 → 종료 후 대화 분석으로 문장 추출 → 발음 채점.

sequenceDiagram
    autonumber
    participant C as 📱 클라이언트
    participant WS as ws_router / call_session
    participant GL as Gemini Live
    participant ST as Supabase Storage
    participant AN as Gemini 분석
    participant SS as SpeechSuper
    participant DB as Postgres

    C->>WS: WS /api/v1/calls/stream (오디오 스트림)
    WS->>GL: 실시간 오디오 릴레이 (system_instruction 주입)
    GL-->>WS: AI 음성 응답 스트림
    WS-->>C: AI 음성 재생
    WS->>ST: 통화 원본 음성 업로드(voice_url)
    WS->>DB: call / call_raw_data 저장 (status=analyzing)

    Note over WS,AN: 통화 종료
    WS->>AN: 대화 전사 분석
    AN-->>WS: 문장(sentence) 추출 + 요약
    WS->>DB: sentence 저장, call.status=done

    C->>SS: 문장 따라 말하기(녹음) → 채점 요청
    SS-->>DB: review(feedback: 점수·글자별 상/중/하) 저장

참고: domains/learning/realtime/, core/gemini_live.py, core/gemini_analysis.py, core/speechsuper.py
